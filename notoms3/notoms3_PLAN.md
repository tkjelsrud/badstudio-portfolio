# Notoms #3 - Summer Canvas

## Guideline

This summer should not be:
- make an EP and prove it matters

This summer should be:
- make 3 strong Notoms tracks in a way that is sustainable

The main scorecard is:
- finished tracks
- strong mood and identity
- a process that does not drain motivation

Finishing and releasing are separate decisions.

## Context

- Project: `Notoms #3`
- Period: `Summer 2026`
- Type: side project
- Current risk: the finishing phase expands and drains the project
- Main protection: keep scope small, finish 3 tracks, then pause
- Alternative composition mode: see `notoms3_komposisjonsworkflow.md`
- Track logs live in per-song files: `tr_*.md`

## Destination

- finish 3 strong tracks
- build a believable Notoms world
- make something personal, atmospheric, and replayable
- end summer with material that can be released later if it still holds up

## Vehicle

- lofi / dub / ambient EP world
- spoken or sung elements
- guitar
- beats from hardware samplers
- synths
- at least 1 track with a guest vocal or guest instrument

## Petrol

- curiosity
- writing and spoken-word fragments
- loop building
- small arrangement experiments
- texture hunting
- the fun of finding a mood that feels true

## Keys

- mood beats polish
- `3 plus overflow`
- only 3 tracks get finishing hours
- extra ideas stay as sketches
- let ideas sit for 1-2 weeks before judging them
- pause after 3 finished tracks, then decide release
- do not judge the project mainly by plays
- use 2-3 music peers as a small feedback circle

## Instruction

Finish 3 strong tracks over a sustainable 6-8 week period.

Allow extra sketches, but do not let them become finishing work unless one of the main 3 clearly fails.

Once 3 tracks are finished:
- pause
- let them sit
- share with 2-3 music peers
- then decide whether to release quickly, wait longer, or build toward a fuller EP later

## Operating Rules

### Capture

- record loops, fragments, riffs, beats, synth textures, and vocal phrases
- keep ideas small at first
- do not force every sketch into a full arrangement immediately

### Track Logs

- each active track gets one running `tr_*.md` file
- use ChatGPT as a practical dialogue partner for the active song
- append short notes after sessions, takes, and useful ChatGPT discussions
- log more than you analyze
- capture what changed, what worked, what failed, what to try next, and any learning worth keeping
- end each entry with the clearest next action

### Maturation Gap

- store ideas for 1-2 weeks
- listen back with fresher ears
- ask whether the idea is still usable, still interesting, and still part of the same world

### Finish Rule

A track counts when:
- the mood lands
- the identity is clear
- it survives distance and re-listening
- it feels worth hearing more than once

It does not need to be maximally polished.

### Anti-Grind Rule

- do not work the same tune for several days in a row if you are only making micro-adjustments
- rotate between sketches
- if repeated tweaking is not making the track clearly stronger, pause it

### Promotion Baseline

Keep promotion real and manageable:
- update artist profile and photo
- align Spotify, Bandcamp, and SoundCloud
- make one honest release post

Optional only if energy is there:
- process posts during creation
- selective sharing to Reddit, Hudd, or Facebook groups
- curator outreach or paid pitching

### Post-Release Protection

To avoid the post-release vacuum:
- decide the next small move before release day
- share the music with a small trusted circle
- do not use play count as the main scorecard

Possible next moves:
- take one week off
- start one new sketch
- write one process note
- make one alternate or live version
